refactor: consolidate duplicate menu/tab navigation components

## Summary
Eliminate code duplication in tabs/menu navigation by creating two reusable,
production-ready components that replace manual implementations across modules.

## Changes

### New Components
- feat(ui): TabsInternal component for state-based tab management
  * Used in forms, wizards, modals where tabs are local state
  * Includes 3 color variants (default, light, dark)
  * Type-safe with TypeScript, ARIA attributes for accessibility
  * Centralized getTabClass() function for consistent styling

- feat(ui): TabsQueryParam component for URL-based tab management
  * Used in detail pages, dashboards where tabs are URL-shareable
  * Query parameters: ?tab=id (or custom param name)
  * Same styling consistency via shared getTabClass()
  * Includes 3 color variants (default, light, dark)
  * Server-compatible with Next.js Link component

### Refactoring
- refactor(employees): Update employee-edit-form to use TabsInternal
  * Removed duplicate tabClass() function (10 lines)
  * Replaced manual button navigation with component (7 lines removed)
  * Added tabs array definition (5 lines)
  * Net: -17 lines of code (consolidated)

- refactor(employees): Update employee detail page to use TabsQueryParam
  * Removed duplicate tabClass() function (10 lines)
  * Replaced manual Link navigation with component (5 lines removed)
  * Added tabs array definition (3 lines)
  * Net: -15 lines of code (consolidated)
